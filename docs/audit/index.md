# Audit

This section contains reusable prompts for checking the project against its research contract. Use these pages when handing work to another coding agent, reviewing implementation state, or checking the future manuscript against available evidence.

## Development Audit

[Development Audit](development.md) is the coding handoff and implementation review prompt. It keeps the engineering work aligned with the paper design, data contract, timestamp rules, and test-coverage expectations.

Use it before adding new pipeline stages or when reviewing whether current code claims more than the available evidence supports.

## Manuscript Audit

[Manuscript Audit](manuscript.md) is the paper-facing review prompt. It checks whether the LaTeX manuscript describes the empirical object, data sources, identification design, model comparisons, and contribution boundaries accurately.

Use it before circulating a draft, adding results tables, or converting engineering artifacts into manuscript claims.
